# Meeting Analysis Checklist

## Extract First

| Item | What To Look For |
|---|---|
| Decision | Explicit agreement, selection, approval, rejection, or direction change |
| Action | Verb + owner + expected output |
| Deadline | Date, meeting, iteration, launch, or review point |
| Concern | Repeated question, pushback, risk, missing evidence, unclear scope |
| Evidence | Data, example, screenshot, research, user feedback, test result |
| Open question | Point not settled in the meeting |

## Concern Categories

| Concern | Signal |
|---|---|
| Result | outcome, value, conversion, cost, quality, satisfaction |
| User path | who uses it, where they enter, what action they complete |
| Execution | owner, time, sample, test, handoff, acceptance |
| Data | metric, source, definition, dashboard, evidence |
| Scope | first version, later version, not included, dependency |
| Risk | permission, privacy, money, security, contract, policy, quality |
| Clarity | people are using different terms or assumptions |

## Practical Inference

Allowed:

- "The material suggests this stakeholder needs a decision option before detail."
- "The next report should start with current result and risk."

Avoid:

- "This person is always..."
- "Their hidden motive is..."
- "They definitely wanted..."

## Next Action Rules

An action item is complete only when it includes:

1. task
2. owner
3. expected output
4. deadline or review point
5. acceptance or next confirmation
