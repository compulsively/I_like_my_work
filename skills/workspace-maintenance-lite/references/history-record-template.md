# History Record Template

Use for meaningful changes that future teammates may need to understand.

## Minimal Format

```markdown
# History: [Topic]

## Latest Context

- Goal:
- What changed:
- Why:
- Impacted files or folders:
- Current decision:
- Follow-up:

## History

### [Date]

- What happened:
- Why it happened:
- What was affected:
- What to watch next:
```

## When To Record

Record when:

- a skill, rule, or shared workflow changes
- folders are reorganized
- an index or README is updated after meaningful structure change
- an old tool or process is archived
- a team decision affects future work

Do not record:

- small typo fixes
- one-off file edits
- temporary experiments that are discarded
