# Backend Scope Rules

## Light

Use for:

- one small backend change
- one field, button, switch, or copy change
- no new permission or workflow

Include:

1. page
2. change point
3. rule
4. acceptance

## Standard

Use for:

- one backend page
- list, detail, form, filter, export, or basic configuration
- limited states and permissions

Include:

1. page purpose
2. operator workflow
3. search/filter/table/form structure
4. field definitions
5. actions and conditions
6. states and exceptions
7. permissions when relevant
8. acceptance criteria

## Complex

Use for:

- multiple pages or modules
- approval, batch operation, import/export, scheduling, or configuration effective rules
- role and data-scope differences
- operational risk or rollback matters

Include:

1. module scope
2. information architecture
3. role and data-scope model
4. workflow and state transition
5. field and configuration rules
6. operation logs
7. exception and conflict handling
8. rollout and acceptance

## Console-Level

Use for:

- full admin console
- multiple modules
- shared permission, menu, dictionary, log, or configuration framework

Add:

- module map
- menu structure
- global permission model
- global dictionary/config rules
- cross-module impact
- admin governance checklist
