# Document Scale Rules

## Light Document

Use for:

- text, button, label, or small configuration change
- single page with clear behavior
- no complicated state or permission

Include:

1. background
2. goal
3. scope and non-scope
4. function description
5. key rule
6. acceptance criteria

## Standard Document

Use for:

- normal feature or page flow
- user-facing or operator-facing behavior
- several states or exceptions
- fields, permissions, or data sources matter

Include:

1. background and goal
2. user roles and scenarios
3. scope and non-scope
4. main flow and branch flow
5. page/module requirements
6. fields and rules
7. states and permissions
8. exceptions
9. acceptance criteria
10. risks and open questions

## Complex Document

Use for:

- multiple roles, pages, or systems
- cross-step state transitions
- permissions, data, approvals, or launch risk
- multiple teams need to implement or test

Include:

1. goal and success criteria
2. roles, scenarios, and responsibilities
3. document scope and release phases
4. global flow and page/module list
5. state machine
6. field table and data source
7. permission matrix
8. exception and conflict handling
9. acceptance criteria and test focus
10. metrics or observation plan
11. launch, rollback, and open decisions
