# User Experience Lite

## Purpose

Add a lightweight user-experience lens to product work before requirements, documents, prototypes, or backend workflows become only system-first.

This skill helps check whether the target user can understand the page, complete the task, recover from problems, and trust the result.

## Load Order

1. Read `references/ux-checklist.md`.
2. Read `references/ux-output-template.md` before producing review comments or improvement suggestions.

## Workflow

1. Identify target user and task.
2. Identify artifact type:
   - requirement
   - document
   - prototype
   - page design
   - backend workflow
   - form or process
3. Check the core path:
   - entry
   - primary action
   - feedback
   - result
   - recovery
4. Find friction:
   - unclear wording
   - too many steps
   - missing state
   - weak priority
   - hidden rule
   - risky action without confirmation
5. Separate evidence from assumptions.
6. Output practical fixes and experience acceptance criteria.

## Output Rules

- Keep suggestions tied to task success, clarity, efficiency, trust, or error recovery.
- Do not invent user research, metrics, test results, or support data.
- Do not use personal taste as product truth.
- Do not weaken security, privacy, payment, approval, or compliance boundaries for convenience.
- For small changes, output a short UX risk check instead of a full UX report.

## Default Output Shape
