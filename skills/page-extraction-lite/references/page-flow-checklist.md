# Page Flow Checklist

Use this when more than one page or state is involved.

## Flow Questions

1. Where does the user enter?
2. What is the first visible page?
3. What is the primary action?
4. What page or state appears after the action?
5. What happens on empty, error, no permission, or cancelled action?
6. What data carries across pages?
7. What state or status changes?
8. Where does the flow end?

## Common Missing Items

- no entry point
- no exit state
- action exists but result is unclear
- status changes are visible but not explained
- page has data but field source is unknown
- error and empty states are missing
- multiple pages have inconsistent naming
