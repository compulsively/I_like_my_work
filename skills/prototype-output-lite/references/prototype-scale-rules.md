# Prototype Scale Rules

## Light Prototype

Use for:

- one page
- one primary action
- simple display or form
- no complicated state transition

Include:

1. page purpose
2. layout blocks
3. primary action
4. basic states
5. small mock data

## Standard Prototype

Use for:

- several states on one page
- normal feature flow
- form validation, filter, list, detail, modal, or drawer
- user path needs review

Include:

1. page list
2. entry and exit
3. layout structure
4. component list
5. interaction path
6. state coverage
7. mock data
8. review focus

## Complex Prototype

Use for:

- multiple pages or roles
- complex flow, approval, payment, status, or operation process
- scenario switching or review demo is needed

Include:

1. flow map
2. page map
3. role or scenario switch
4. state machine
5. edge and error cases
6. review script
7. handoff notes
8. open decisions
