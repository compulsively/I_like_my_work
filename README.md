# I Like My Work

Lightweight skill pack for common product workflow tasks:

1. requirement intake
2. document output
3. prototype output
4. page extraction
5. general audit
6. workspace maintenance
7. local context setup

## Suggested Use Order

1. Install or copy the folders under `skills/` into your own Codex skills directory.
2. Start with `local-landing-context-template-pack` and fill your team's local context.
3. Use `requirement-intake-lite` when a request is unclear.
4. Use `document-output-lite` for PRDs, feature notes, and acceptance criteria.
5. Use `prototype-output-lite` for page structures, prototype briefs, and prototype prompts.
6. Use `page-extraction-lite` to turn screenshots or page notes into structured page docs.
7. Use `general-audit-lite` to check whether a work output roughly meets expectations.
8. Use `workspace-maintenance-lite` for indexes, archive candidates, cleanup candidates, and history records.

## Boundary

This pack is generic. It does not include private company context, personal workflow details, specific stakeholder interpretation, domain-specific compliance rules, or local filesystem paths.

Each team should fill its own local context before relying on the output for real projects.
